# 客户端需求文档 v3

> 更新日期：2026-04-25  
> 更新依据：当前 Qt 前端实现、`Server/DataBaseDesign.md` v2、`Server/DatabaseER.mmd`、`PRODUCT_OPTIMIZATION_NOTES.md`。  
> 适用范围：Qt 客户端优先，覆盖桌面端与移动端两种形态；Web/管理端可复用同一套后端接口，但不在本文展开。

## 0. 本次更新目标

- 分析当前 Qt 前端已经实现的页面、数据流和缺口。
- 在不推翻现有 Qt Widgets 代码基础上，补齐桌面端横屏和移动端竖屏的界面架构。
- 参考 GitHub Primer 的侧边导航、响应式分栏与窄屏单列原则，结合 Quizlet、Anki、Duolingo 等学习产品的学习路径、复习限制、打卡与排行榜设计，形成适合本项目的 Qt UI 规范。
- 对齐后端新设计：账号密码登录、词书/题库/阅读/听说/口语/统计/小组/同步等服务端实体和客户端本地 SQLite 子集。

## 1. 当前 Qt 前端现状分析

### 1.1 已有实现

- `Client_QT/EnglishLearner/EnglishLearner.ui` 基本还是空的 `QMainWindow` 壳，实际主界面由 `EnglishLearner.cpp` 代码动态构建。
- 主窗口当前尺寸为 `1120 x 760`，最小尺寸为 `980 x 660`，天然偏桌面横屏。
- 当前主框架使用顶部信息条 + 底部 `QTabWidget` 导航，包含“单词、听说、阅读、我的”四个主模块。
- 登录弹窗 `LoginDialog.cpp` 已实现服务地址输入、手机号/微信/QQ/邮箱入口、注册、登录、离线体验和忙碌状态。
- 单词模块已有词书选择、每日新词/复习数量、保存计划、今日概览、开始学习/复习、四选一答题、答题反馈、本地事件记录。
- 阅读模块已有左侧材料列表 + 右侧详情的桌面分栏。
- 听说模块已有听力材料和口语任务双栏，包含进度、文本稿、口语本地评测模拟。
- 我的模块已有学习统计、打卡、分享包、小组/同桌、排行榜、同步状态。
- `LocalStore` 已按用户打开 SQLite，保存本地用户、词书、计划、学习事件、答题记录、今日汇总、阅读/听力/口语、小组、打卡和 `local_sync_outbox`。
- `ApiClient` 已接入认证、当前用户、下一题、提交答案、排行榜等基础接口。

### 1.2 主要缺口

- 桌面端和移动端没有明确区分：当前底部 Tab 更像移动导航，但窗口尺寸和分栏更像桌面软件。
- 主窗口最小宽度较大，不适合竖屏移动端或小屏平板。
- 登录页展示了手机号、微信、QQ、邮箱入口，但后端当前正式启用的是账号密码登录；其他登录方式应作为预留入口，不应表现为已完整开放。
- 当前同步按钮只是模拟清空待同步队列，文档需要明确真实同步状态和失败重试交互。
- 学习状态、错题、收藏、同步、设置等入口比较分散，需要统一成“当前上下文动作 + 全局侧边导航/底部导航”。
- 题库、错题、收藏、阅读题、听力题、口语评测等后端新模型已有设计，但 Qt UI 只做了演示级入口。
- `EnglishLearner.ui` 与实际代码不一致，后续若继续用 Qt Widgets，应明确以代码构建或 Designer 构建为主，避免双来源漂移。

## 2. 参考产品与设计原则

### 2.1 外部参考

- GitHub Primer：`NavList` 用于当前上下文的垂直导航，并清晰指示当前视图；复杂列表可分组，条目可带图标和辅助信息。响应式布局建议把桌面多列能力和窄屏单列体验分开，窄屏下将列表详情拆成可钻取页面。
- Quizlet Learn：学习流程强调“选择学习集 -> 选择目标 -> 进入个性化学习路径”，并允许用户配置题型、音频、收藏词和评分方式。
- Anki：每日新卡上限和每日复习上限是长期记忆负担控制的核心，复习积压时不应继续无限引入新词。
- Duolingo：打卡、连胜、排行榜和好友任务用于维持学习动机，但应让用户可控，避免竞争压过学习主任务。

### 2.2 本项目落地原则

- 学习主任务优先：首屏突出“今天该学什么、还剩多少、立即开始”，激励数据放在次级区域。
- 桌面端信息密度更高：侧边导航、列表详情、右侧状态栏并存，减少页面跳转。
- 移动端路径更短：底部导航 + 单列页面 + 钻取详情 + 底部操作栏，优先单手操作。
- 组件风格统一：不直接依赖系统默认控件外观，使用统一颜色、圆角、边框、图标、状态标签和进度条。
- 像素风只作为品牌点缀：可用于图标、空状态、每日插图或背景，不影响长文本阅读和答题清晰度。
- 所有学习动作必须有状态反馈：加载中、离线、同步中、已同步、同步失败、保存成功、答题正确/错误、积分变化。

## 3. 客户端定位

- Qt 客户端是主要学习端，同时支持桌面端和移动端。
- 客户端采用“离线优先、最终同步”的学习体验：核心学习内容、学习计划、答题记录、阅读/听力/口语进度需要能在本地 SQLite 中保存，联网后再同步到服务端。
- 客户端覆盖用户侧能力：账号登录、词书与单词计划、答题与错题、阅读理解、听力口语、学习统计、打卡积分、排行榜、小组/同桌、离线同步。
- 管理后台相关能力只在管理员账号下展示入口；普通学习用户不展示角色/权限信息。

## 4. 视觉与交互风格

- 应用品牌名沿用当前代码中的 `LingoPulse`。
- 视觉关键词：清晰、轻量、可持续学习、有一点像素风趣味。
- 颜色建议使用中性色底 + 蓝绿色学习主色 + 橙色激励点缀 + 红色错误反馈，避免整套界面只依赖单一蓝紫渐变。
- 浅色主题默认背景为低饱和灰蓝；深色主题默认背景为深灰蓝，卡片和输入框要保证足够对比度。
- 卡片圆角建议控制在 8-12px；学习主卡、弹窗可略大，但不把所有页面区域都做成卡片堆叠。
- 按钮分级：主操作、次操作、文本/幽灵操作、危险操作；学习页主操作固定在视觉中心或底部安全区。
- 交互控件要求：
  - 桌面按钮最小高度 36px，移动按钮和可点击列表项建议 44px 以上。
  - 移动端底部导航图标 + 短标签，最多 5 项。
  - 桌面端侧边导航图标 + 标签 + 可选计数徽标。
  - 筛选、难度、题型用分段控件或下拉；数值目标用步进器；开关项用 Switch/CheckBox。

## 5. 响应式布局规范

### 5.1 断点与形态

| 形态 | 建议判断 | 导航 | 页面结构 |
| --- | --- | --- | --- |
| 移动竖屏 Compact | 宽度 `< 768px` 或竖屏手机 | 底部导航 | 单列、钻取详情、底部操作栏 |
| 平板/小桌面 Regular | 宽度 `768-1199px` | 可折叠侧边栏或顶部轻导航 | 双列优先，必要时局部堆叠 |
| 桌面横屏 Wide | 宽度 `>= 1200px` | 左侧固定侧边栏 | 多列仪表盘、列表详情、右侧状态栏 |

Qt 实现建议提供 `AdaptiveShell` 或等价布局控制层：监听窗口尺寸、方向和平台类型，在不重启页面状态的情况下切换导航和分栏。

### 5.2 桌面端横屏设计

桌面端默认使用左侧侧边导航，宽度建议 220-260px；窄桌面可折叠为 64-72px 图标栏。

侧边导航分组：

- 学习：今日、单词、练习。
- 内容：阅读、听说、收藏。
- 社区：小组/同桌、排行榜。
- 个人：统计、我的、设置/同步。

桌面页面结构：

- 顶部上下文栏：当前页面标题、当前词书/计划、搜索、同步状态、用户头像。
- 左侧导航：全局模块切换。
- 主内容区：学习任务、列表详情或正文。
- 右侧状态栏：今日目标、待复习、待同步、最近错题、快捷动作；当窗口不足时收起为按钮或抽屉。

桌面典型布局：

- 今日页：左侧今日任务队列，中间学习进度与开始按钮，右侧积分/连续天数/同步状态。
- 单词页：左侧词书和计划，中间复习队列或学习卡，右侧单词状态、错题、收藏。
- 阅读页：左侧文章列表与筛选，中间阅读正文，右侧进度、题目、笔记和收藏。
- 听说页：左侧材料列表，中间播放器/录音器，右侧文本稿、评分详情和历史记录。
- 我的页：三栏仪表盘，分别展示统计、社区/排行榜、账号与设备同步。

### 5.3 移动端竖屏设计

移动端默认底部导航，建议 4-5 个主入口：

- 今日：任务、连续天数、今日进度、开始学习。
- 单词：词书、计划、学习/复习队列。
- 练习：题库、错题、收藏。
- 内容：阅读、听力、口语。
- 我的：统计、小组、排行榜、账号、同步。

移动页面规则：

- 单列优先，列表和详情分成两个页面；进入详情后顶部提供返回。
- 学习和答题页面全屏沉浸，底部固定“认识/不认识/下一题/提交”等操作。
- 阅读正文使用全屏阅读器，题目、笔记、收藏、进度从底部抽屉打开。
- 听力播放器固定在底部，文本稿可滚动；口语录音按钮固定在底部安全区。
- 同步失败、离线、待上传等状态用顶部轻提示或我的页状态卡展示，不打断学习。
- 桌面右侧状态栏在移动端改为“今日状态”卡或底部抽屉。

## 6. 账号、登录与用户资料

对应服务端表：`users`、`user_auth_identity`、`user_profile`、`user_device`、`email_verification`、`notification_message`。

### 6.1 登录页面

- 当前正式登录方式：账号密码登录，接口优先使用 `POST /api/v1/auth/login/password`，请求体为 `account` + `password`；`POST /api/v1/auth/login` 仅作为兼容别名。
- 注册接口继续使用账号 + 密码首期流程；账号长度、密码长度和错误提示需要与后端校验一致。
- 登录页主入口显示“账号/邮箱登录”；手机号、微信、QQ 作为“即将支持/绑定预留”入口，不默认暗示已完整接入。
- 支持离线体验，但离线用户需要显式标识为本地演示账号，避免与正式账号同步混淆。
- 登录成功后保存 JWT、用户资料和设备信息，生成或复用 `device_uuid`，记录 `platform`、`app_version`、`last_seen_at`。
- 桌面登录页：居中登录卡 + 左侧或顶部品牌信息，最大宽度约 420-480px。
- 移动登录页：全屏单列，服务器地址折叠到“高级设置”，主操作按钮固定在表单下方。

### 6.2 用户资料与设置

- 展示头像、昵称、账号、邮箱、手机号、英语水平、目标水平、每日学习时长、时区。
- 支持修改学习偏好，对应 `user_profile.english_level`、`target_level`、`daily_minutes`、`timezone`、`preference_json`。
- 展示当前设备列表和同步状态。
- 支持退出登录；退出时本地数据按用户隔离，避免不同账号学习记录混用。
- 管理员账号可在设置页展示“管理后台入口”；普通用户不展示角色和权限细节。

## 7. 应用主框架与导航

### 7.1 信息架构

- 今日：聚合今日目标、待复习、新词、阅读/听力/口语任务、连续天数、同步状态。
- 单词：词书、计划、学习、复习、单词状态。
- 练习：统一题库、错题、收藏、专项测验。
- 内容：阅读、听力、口语。
- 社区：小组、同桌、排行榜、分享包。
- 我的：统计、成就、积分、账号、设备、同步、设置。

桌面端可全部展示在侧边栏；移动端底部导航只保留高频入口，将社区和设置放入“我的”。

### 7.2 导航状态

- 每个模块右上或上下文栏保留“学习状态”入口，用于查看当前进度、历史记录、错题、收藏和同步状态。
- 侧边栏或底部导航需要展示关键计数：待复习、错题、待同步。
- 当前页面要有明确选中态；桌面侧边栏允许二级导航，但最多两层。

## 8. 今日与单词板块

对应服务端表：`book`、`book_word`、`vocabulary_word`、`user_word_plan`、`user_word_state`、`word_study_event`、`daily_study_summary`、`score_transaction`。

### 8.1 今日页

- 今日页是打开应用后的默认页。
- 展示今日一句或每日单词，背景可使用轻量插图或像素风图块，但不能影响文字可读性。
- 展示今日学习概览：新学单词数、复习单词数、正确率、学习时长、连续学习天数、今日积分变化、待同步数量。
- 生成“今日任务队列”：单词复习、新词学习、阅读继续、听力继续、口语任务。
- 主按钮根据队列自动决定文案：继续复习、开始新词、继续阅读、同步后开始等。
- 桌面端在右侧显示今日状态栏；移动端把今日状态做成首屏卡片，主按钮放在首屏下方。

### 8.2 词书与计划设置

- 展示可用词书列表，包括系统内置词书和用户本地上传词书。
- 词书元数据对应 `book`，词书单词关系对应 `book_word`，单词基础信息对应 `vocabulary_word`。
- 支持选择当前学习词书，并创建或切换 `user_word_plan`。
- 支持设置每日新学数量、每日复习数量、复习算法类型。
- 计划设置需要提醒学习负担：当待复习积压过多时，建议降低每日新词数量。
- 展示当前词书进度：总词数、已学、待复习、已掌握、易错词。
- 用户自行上传书籍时，正文暂时仅保存在客户端本地；服务端默认只保存 `book` 元数据、`client_content_ref`、`content_hash`。
- 支持客户端之间分享传输自上传内容，分享时携带书籍元数据、内容校验值和必要的本地内容包。

### 8.3 单词学习页面

- 展示单词、英/美音标、发音按钮、释义、词性、例句、难度。
- 首期保留四选一题型；后续支持认识/不认识、拼写、听音辨词、例句填空。
- 选择题选项优先来自 `question_option`；单词题也可以由 `vocabulary_word` 临时生成。
- 每次学习、复习、答题都生成一条 `word_study_event`，离线时先写入本地待同步队列。
- 答题后即时更新本地 `user_word_state`：正确次数、错误次数、掌握度、下次复习时间。
- 支持收藏当前单词或题目，对应 `favorite_item`。
- 答错题目需要进入错题集，对应 `wrong_answer`。
- 桌面端学习卡居中，右侧显示词书进度和复习队列。
- 移动端学习卡全屏，选项纵向排列，底部固定下一题或提交按钮。

### 8.4 单词状态页

- 展示当前计划的学习状态：学习中、待复习、已掌握、易错词。
- 展示复习队列、最近学习事件、今日完成情况。
- 展示日历或趋势图，数据来自 `daily_study_summary`。
- 支持按同步状态筛选：本地未同步、同步失败、已同步。

## 9. 题库、错题与收藏

对应服务端表：`question`、`question_option`、`question_attempt`、`wrong_answer`、`favorite_item`。

- 所有模块的答题记录统一写入 `question_attempt`。
- 题目支持单词题、阅读理解题、听力题、口语题等模块类型，对应 `question.module_type`。
- 答题后展示正确答案、解析、得分变化和耗时。
- 错题集按模块筛选：单词、阅读、听力、口语。
- 错题复习答对后可标记为已解决，对应 `wrong_answer.status = RESOLVED`。
- 收藏夹支持按类型展示：单词、题目、阅读文章、听力材料、口语任务。
- 收藏夹预留文件夹名称，对应 `favorite_item.folder_name`。
- 桌面端错题列表和题目详情并排；移动端先列表后详情，复习时进入全屏答题。

## 10. 阅读理解板块

对应服务端表：`reading_passage`、`reading_progress`、`question`、`question_attempt`。

- 展示阅读文章列表，支持按难度、词数、预计阅读时长、来源筛选。
- 阅读内容来源包括管理后台统一录入和用户自行上传；用户上传内容处理策略同书籍正文，暂时仅保存在客户端本地。
- 阅读详情页展示标题、正文、摘要、难度、预计阅读时间。
- 需要记录阅读进度：百分比、最后阅读位置、阅读秒数、完成时间。
- 阅读理解题通过 `question.source_type = READING_PASSAGE`、`question.source_id = reading_passage.id` 关联。
- 阅读页支持收藏文章、继续阅读、完成后答题。
- 桌面端采用左列表 + 中正文 + 右进度/题目/笔记。
- 移动端采用文章列表 -> 阅读器 -> 题目页的三段钻取流程。
- 离线阅读进度需要写入本地，联网后同步。

## 11. 听力与口语板块

对应服务端表：`listening_material`、`listening_progress`、`speaking_task`、`speaking_attempt`、`media_asset`。

### 11.1 听力

- 展示听力材料列表，支持按难度、时长、来源筛选。
- 听力详情页包含标题、音频播放器、文本稿、学习进度。
- 记录听力进度：已听秒数、完成次数、最后播放位置。
- 听力题可复用统一题库和答题记录。
- 桌面端播放器和文本稿并排；移动端播放器固定在底部，文本稿单列滚动。

### 11.2 口语

- 展示口语任务列表，包含提示文本、参考文本、难度。
- 支持录音、回放、提交评测。
- 口语任务来源包括管理后台统一录入和用户自行创建，用户自行上传内容处理策略同书籍正文。
- 首期可以先保存录音和作答记录；第三方语音评测 SDK 接入后，将识别文本、总分、细项分数写入 `speaking_attempt`。
- 语音评测预留本地评测和服务端评测两种接入方式，用户录音暂时不强制上传服务端。
- 音频资源对应 `media_asset`；若后续启用服务端评测或云端保存，再上传到对象存储。
- 移动端录音按钮固定在底部安全区，并清楚展示录音、回放、重录、提交四种状态。

## 12. 学习统计、打卡、社区与激励

对应服务端表：`daily_study_summary`、`checkin_record`、`score_transaction`、`achievement`、`user_achievement`、`study_group`、`study_group_member`、`study_partner`、`leaderboard_snapshot`、`users.total_score`。

- 我的页面展示今日学习汇总、连续学习天数、总积分、已获得成就。
- 支持每日打卡，打卡后展示连续天数和奖励积分。
- 展示积分流水，说明积分来源：答题、复习、打卡、成就奖励。
- 排行榜支持全站榜、词书榜、小组榜、好友/同桌榜，以及日榜、周榜、月榜、总榜。
- 排行榜数据以 `leaderboard_snapshot` 为准，小规模实时榜可参考 `users.total_score`。
- 客户端支持用户自建学习小组或建立同桌关系，用于共同学习、进度对比和互相激励。
- 激励设计要克制：排行榜和小组入口不抢占今日学习主按钮；移动端将社区入口收进“我的”或“今日状态”。
- 排行榜需要提供隐私控制入口，允许用户退出公开榜或只参与小组榜。

## 13. 离线数据与同步

对应服务端表：`user_device`、`sync_task_log`、`sync_change_item`、`server_change_log`，对应客户端本地表：`local_sync_outbox`、`local_sync_meta`。

- 客户端首次登录后生成并保存 `device_uuid`。
- 所有离线创建的学习事件都需要生成稳定的 `client_event_id`，重试上传时保持不变。
- 本地数据同时保存 `local_id` 和 `server_id`：`local_id` 用于离线立即可用，`server_id` 用于同步成功后的服务端引用。
- 客户端上传同步时携带 `device_uuid`、`client_event_id`、`client_updated_at`。
- 服务端以 `user_id + client_event_id` 做幂等去重。
- 普通学习记录冲突采用“最后修改时间优先”；账号、安全、权限类数据采用“服务端优先”。
- 删除操作统一按逻辑删除处理，并同步 `is_deleted = 1`。
- 网络恢复后自动尝试上传 `local_sync_outbox` 中的待同步变更，并拉取 `last_pulled_version` 之后的服务端增量。

同步 UI 状态：

- 已同步：绿色状态点或轻量标签。
- 待同步：显示待上传数量，不打断学习。
- 同步中：显示进度或旋转状态，禁止重复触发。
- 同步失败：显示失败原因、重试按钮和最后一次失败时间。
- 冲突需处理：进入同步详情页，按实体显示“使用本地/使用服务端/稍后处理”。

## 15. Qt 实现建议

- 主界面新增自适应壳层，替代单一 `QTabWidget` 底部导航：
  - 桌面端：`QSplitter` 或 `QHBoxLayout` 组织侧边栏、主内容区、右侧状态区。
  - 移动端：保留底部导航，主内容使用 `QStackedWidget` 管理钻取页面。
- 保留现有模块函数，但拆出页面级组件：`TodayPage`、`WordPage`、`PracticePage`、`ReadingPage`、`ListeningSpeakingPage`、`MinePage`、`SyncPage`。
- 将“学习状态”从弹窗升级为可复用状态面板：桌面为右侧面板，移动为底部抽屉或独立页面。
- 将登录方式展示改为“账号登录”为主，第三方登录入口置灰或标注预留。
- `EnglishLearner.ui` 若继续使用 Designer，应迁移真实布局；若继续代码构建，应在文档中约定 `.ui` 只作为空壳，不再重复维护。
- 引入统一样式表和组件工厂：按钮、状态标签、导航项、学习卡、列表项、空状态、错误提示。
- 为移动端提供窗口尺寸测试配置，至少验证 390x844、430x932、768x1024；桌面至少验证 1280x800、1440x900。

## 16. MVP 优先级

### P0 必做

- 账号密码注册、登录、登录态保存，登录页与后端当前正式登录方式一致。
- 自适应主框架：桌面侧边栏，移动底部导航。
- 今日页、词书选择、单词计划设置。
- 单词学习、复习、答题、学习事件记录。
- 本地 SQLite 保存当前用户、词书、单词计划、学习事件、答题记录和今日汇总。
- 基础同步：离线事件上传、同步状态展示、失败重试。
- 我的页面展示今日学习汇总、积分、连续学习天数。

## 17. 已确认设计决策

- 用户自行上传内容暂时仅保存在客户端，预留保存到服务端或对象存储的接口；客户端之间需要支持分享传输。
- 阅读、听力、口语内容采用“后台统一录入 + 用户自行上传”双来源；用户上传内容处理方式同书籍。
- 排行榜数据在后台存储，范围包括全站榜、词书榜、小组榜、好友/同桌榜、日榜、周榜、月榜、总榜。
- 普通客户端不展示角色/权限信息，管理员账号才展示管理入口。
- 语音评测预留本地评测和服务端评测两种方案，用户录音暂时不强制上传服务端。
- Qt 桌面端优先侧边栏与横向分栏，Qt 移动端优先底部导航与竖向单列。
