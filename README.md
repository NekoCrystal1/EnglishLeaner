# 交互式英语自学系统

## 项目部署

### 部署架构

本系统采用 C/S + B/S 混合架构，支持桌面客户端和 Web 浏览器两种访问方式，后端服务统一提供 API 接口。
Qt 客户端 & Web 浏览器
*-访问->
*Nginx 反向代理(负载均衡 / 静态资源)
-调用->
Java Spring Boot 后端服务
-读写->
Sql Server数据库（可实现云端存储）

### 部署方式

| 组件  | 部署环境 | 说明 |
| --- | --- | --- |
| 后端服务 | 云服务器/本地 | Java |
| 数据库 | 云服务器/本地 | SQL Server |
| Web 前端 | 对象存储 OSS + CDN | 静态资源托管，加速访问 |
| Qt 客户端 | 用户本地安装 | 提供安装包下载 |

## 项目主要功能展示

### 核心功能

- **交互性：** 不仅仅是展示，还包含大量练习、即时反馈。
- **多模态学习：** 整合文本、音频、视频、图像等多种学习资源。*
- **数据驱动：** 利用学习行为数据进行学习分析，为用户提供可视化报告或改进建议。

### 特色功能

- **比较心理学应用：** 利用排行榜等机制激发积极性
- **LearnGap：** 可以与同桌、同组或者在线匹配其他用户，进行一场交互式学习pk，如相互口语练习，单词学习pk
- **打卡激励：** 提供打卡正反馈，提高持续学习积极性
- **成就系统：** 提高学习产生的结果正反馈，进而提高积极性
- **与ai学习：** *服务端接入ai，可以与ai进行对话练习、测评等

## 技术选型

### 整体架构

采用 C/S 与 B/S 混合架构，Qt 桌面客户端作为主力学习端，支持完整的离线学习功能；Web 浏览器端提供轻量级访问入口，方便用户快速查看学习进度；Java 后端统一提供 RESTful API 服务，SQL Server 作为云端主数据库，Qt 客户端本地使用 SQLite 实现数据缓存和离线支持。

### 客户端技术栈

 - 使用QT 6.6.3 作为跨平台C++应用开发框架，负责构建客户端
 - SQLite 作为本地嵌入式数据库存储用户记录、缓存数据，以支持完整学习体验

### Web端技术栈

 - Vue3 作为JavaScript前端框架，采用 Composition API 提升代码组织和复用性。Vite 作为构建工具，提供快速的开发热更新和生产构建优化
 - Element Plus 作为 UI 组件库，提供丰富的企业级组件，用于快速搭建管理后台界面。Axios 处理 HTTP 请求，统一配置拦截器实现 Token 认证和错误处理
 - Pinia 作为状态管理方案，管理用户信息、学习进度等全局状态。Vue Router 负责前端路由管理，实现单页面应用的无刷新导航
 - ECharts 用于数据可视化展示，实现学习趋势图、技能雷达图、学习热力图等专业图表

### 后端技术栈

 - 使用 Java 17 作为后端编程语言，Spring Boot 3.x 作为核心应用框架
 - Spring Web 提供 RESTful API，统一返回格式和错误码
 - Spring Security + JWT 实现认证与权限控制（用户端 / 管理端分权）
 - Hibernate Validator 做参数校验，避免无效请求进入业务层
 - MyBatis-Plus（或 JPA）作为数据访问层，按模块拆分 Repository/Mapper
 - Redis 预留为缓存层（验证码、热点数据、限流计数等）
 - 按 Controller -> Service -> Repository 三层结构实现，方便扩展与维护

### 数据库技术栈

 - 使用 SQL Server 作为主数据库，可部署在云端，保存核心、权威数据，定期备份
 - SQLite 作为 Qt 客户端本地数据库，存储用户离线数据，与云端进行同步
 - 服务端使用 Flyway 管理数据库版本迁移（DDL/DML 脚本可追踪、可回滚）
 - 数据访问层采用“SQL 方言隔离”策略，关键 SQL 不与业务逻辑耦合，便于后续切换 MySQL/PostgreSQL

#### 数据库可修改预留设计

 - 所有核心表统一预留基础字段：`id`、`created_at`、`updated_at`、`version`、`is_deleted`
 - 业务表预留扩展字段：`ext_json`（JSON 扩展信息）、`remark`（备注）
 - 用户、课程、题目、学习记录等主表使用“稳定主字段 + 可扩展 JSON”组合，新增需求优先走 `ext_json`，避免频繁改表
 - 外键与索引按“必需最小集合”设计，非核心分析字段通过二级索引后置补充，降低前期结构锁死风险
 - 所有字段变更走迁移脚本，不直接手改线上库，确保开发/测试/生产结构一致
 - 通过配置项预留多数据库驱动：`spring.datasource.driver-class-name`、`spring.jpa.database-platform` 可按环境替换

### 数据同步机制

 - 采用“客户端离线优先 + 服务端最终一致”同步策略
 - 客户端记录本地变更日志（新增/更新/删除），按批次上传服务端
 - 服务端为学习记录维护 `sync_version` 与 `updated_at`，用于增量下发
 - 冲突处理规则：同字段冲突默认“最后修改时间优先”，关键数据（如账户信息）改为“服务端优先”
 - 同步接口支持断点续传与幂等重试（同一批次重复提交不产生脏数据）
 - 同步过程记录 `sync_task_log`，用于失败追踪与人工排查

### 第三方集成

 - 对象存储：预留 OSS/S3 接口用于音视频、图片、题库附件上传与访问
 - 消息通知：预留短信/邮件服务，用于注册验证码、系统通知、找回密码
 - OAuth 登录：预留 QQ/微信/Google 第三方登录接入能力*
 - 语音评测：预留第三方语音测评 SDK 接口（作为后续增强功能）

### 部署运维

 - 环境拆分：`dev`、`test`、`prod` 三套配置，数据库与密钥按环境隔离
 - Nginx 统一入口：处理 HTTPS、静态资源、反向代理与限流
 - 应用部署支持 Docker 化，便于云服务器快速拉起和横向扩容
 - 日志体系：应用日志、访问日志、同步任务日志分离，支持按天归档
 - 数据库运维：每日自动备份 + 关键表定期校验，支持故障回滚
 - 监控告警：预留 Prometheus + Grafana 接入点，监控接口耗时、错误率、数据库连接池状态
